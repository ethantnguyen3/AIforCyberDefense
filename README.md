## Chosen Project Domain
 AI for Cyber Defense

## Project Description
This project designs and proposes a theoretical AI-driven Security Operations Center (SOC) framework that reduces analyst alert fatigue and improves detection of stealthy, multi-stage attacks. It focuses on safer and more reliable AI use in a SOC environment by improving prompt quality, limiting unsafe output behavior, and encouraging evidence-based analysis.

## Tools Used
- Python
- Prompt Engineering Frameworks
- SOAR (Security Orchestration, Automation, and Response) Playbooks


## Weekly Progress Summary
- **Week 1:** Project scoping, requirement gathering, and initial repository structure setup.
- **Week 2:** Developed baseline controls, policy engines, and initial AI prompt templates.
- **Week 3:** Built the defensive pipeline, input filters, and output validators to act as AI guardrails.
- **Weeks 4:** Conducted testing, compiled baseline vs. controlled results, and documented risk mitigation strategies.
- **Week 5:** Finalized evidence appendix, presentation slides, final report, and cleaned up codebase merge conflicts.

## Final Outcome Summary
The final outcome is a robust, theoretical framework for an AI-enhanced SOC. The project successfully produced secure prompt templates, output guardrails, logging schemas, and SOAR playbooks. Validation evidence demonstrates how these AI guardrails effectively reduce unsupported conclusions and improve the reliability of automated security analysis.
